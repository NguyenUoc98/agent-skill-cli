# QA Skills for Effective Test Case Design in Antigravity (ISTQB-Aligned)

This document describes the knowledge areas and skills required for a QA engineer to design effective and efficient test cases according to ISTQB practices.

---

# 1. Requirement Analysis Skill

A tester must be able to:
- Understand business requirements and acceptance criteria
- Identify implicit and explicit rules
- Extract **test conditions** from the test basis

This skill ensures that no critical behavior is left untested.

---

# 2. Application of Test Design Techniques

A competent tester must know when and how to apply:

- Equivalence Partitioning
- Boundary Value Analysis
- Decision Table Testing
- State Transition Testing

Using these techniques helps reduce redundant test cases while maintaining coverage.

---

# 3. Risk Analysis and Prioritization

A tester must be able to:
- Identify components with high failure probability
- Evaluate business impact of failures

This enables effective **risk-based testing** and optimal test effort allocation.

---

# 4. Analytical and Logical Thinking

Testers must be able to:
- Decompose complex requirements into smaller, testable conditions
- Identify hidden constraints and dependencies between fields or actions

---

# 5. Negative Testing Mindset

A skilled tester actively attempts to break the system by:
- Providing invalid or malformed inputs
- Executing actions in incorrect order
- Attempting unauthorized operations

This increases defect detection effectiveness.

---

# 6. Data Domain Knowledge

Testers must understand:
- Valid data formats and ranges
- Data boundaries and constraints
- Data persistence and transformation across system layers

This supports effective boundary and partition-based testing.

---

# 7. State and Workflow Modeling

For systems with workflows or lifecycle states, testers must:
- Identify all system states
- Map allowed and forbidden transitions
- Design tests that cover these transitions

---

# 8. Defining Clear Test Oracles

Testers must define precise expected results that allow objective determination of:
- Pass
- Fail

Without clear test oracles, test execution becomes subjective and unreliable.

---

# 9. Maintaining Traceability and Documentation

Testers must ensure:
- Test cases are linked to requirements
- Test cases are updated when requirements change
- Historical changes are trackable

This supports auditability and regression testing.

---

# 10. Communication and Collaboration

A tester must:
- Ask clarification questions when requirements are ambiguous
- Communicate defects clearly with reproducible steps
- Collaborate with developers, product owners, and designers to refine requirements

---

# 11. Continuous Learning and Improvement

Testers should continuously improve their skills in:
- New testing techniques
- Domain knowledge
- Automation tools and practices

This ensures long-term test effectiveness and quality growth in the project.
